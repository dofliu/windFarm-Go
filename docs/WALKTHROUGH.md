# 離岸風場・運維傳說 — 遊戲攻略 & 教學說明

> 給教師備課/做簡報、給學生理解玩法與修課目標。
> 本文分「**目前已實作**」與「**設計藍圖（未實作）**」兩類，請勿把藍圖當成現況。
> Last updated: 2026-06-21。對應原始碼：`src/`、設計來源：[GAME_DESIGN.md](GAME_DESIGN.md)。

---

## 1. 一句話與學習目標
玩家經營一支離岸風場**運維（O&M）團隊**：調度船隊與技師、備齊備品、在天氣窗內**登塔排除故障**，維持風場**可用率與發電量**。**故障診斷隨堂測驗 + SOP** 是教學核心。

**學生學到什麼**
- 離岸風機常見故障的**首要排查邏輯**（診斷思維，而非死背）。
- O&M 的**現場限制**：天氣窗、船舶耐海象、備品前置期、停機成本、人力科別。
- 用 **KPI（可用率 / 發電量 / 成本）** 衡量運維績效。

---

## 2. 核心循環（出勤就緒閘門）
**維修畫面不能直接進入**，必須完成「出勤就緒檢查」並航行抵達機組：

```
母港大廳 → 調度中心接工單
  → 出勤就緒檢查（缺一不可）：
       ☐ 船舶可用   ☐ 對應科別技師   ☐ 工具等級   ☐ 必備備品在庫   ☐ 天氣窗允許出航
  → 出航（航行耗時）→ 抵達登塔
  → 維修：診斷測驗 + SOP，在「作業窗」倒數內完成
  → 結算：可用率↑、發電量恢復、扣成本、KPI 更新、圖鑑解鎖
  → 故障未修期間，每天損失發電（停機成本）
```

## 3. 介面導覽
**四個畫面**（上方分頁切換）：母港大廳 / 備品交易所 / 出海航行 / 維修作業。

**母港大廳版面**
- **左上「設施」**：調度中心、技師公會、機具工坊、備品交易所、CTV 整備廠、圖鑑、排行（各列顯示即時資訊）。
- **左下「風場動態」**：海象 / 可用率 / 發電量 / 作用中警報 ＋ 輪播**班級動態**（讀雲端排行榜其他玩家分數）。
- **中央**：風場風景 ＋ 大「出海航行」鈕、靠港休整。
- **右側「我的營運」（可收合）**：待處理工單（含接單/下一關）、船隊、技師名單、備品庫存、KPI。
- **上方 HUD**：海象、天數、技師在勤、預算、登入帳號、登出。

## 4. 起始狀態與經濟（目前數值）
| 項目 | 起始值 | 說明 |
|---|---|---|
| 預算 | ◎ 8,420 萬 | 頂部以「萬」顯示 |
| 天數 | 第 21 天 | 日曆制 |
| 可用率 | 86% | 主 KPI |
| 技師在勤 | 24 / 30 | 起始 1 名機械技師「阿銘」 |
| 船舶 | CTV（耐海象 Lv1） | 可到「警戒」海象；SOV 可到「停航」 |
| 每日發電 | 100% 可用率 = 120 MWh/日 | 隨可用率比例累積 |

**收入**：完成工單報酬（15–30 萬/關）、調度中心例行小任務、（選）備品價差。
**支出**：備品採購、招募技師（300 萬 × 等級）、工具升級（(等級+1)×100 萬）、船舶整備（(等級+1)×100 萬）、購置 SOV（3,000 萬）、**停機損失（每天 3 萬）**。

**綜合績效分（排行榜）** = `發電量(MWh) + 可用率×5 + 完成任務×30`

## 5. 故障題庫（教學重點，共 5 種）
| 故障 | 科別 | 必備備品 | 診斷「首要排查」正解 | 知識點 |
|---|---|---|---|---|
| 齒輪箱油溫過高 | 機械 | 齒輪箱齒輪油 | 潤滑油油位與油質 | 潤滑/散熱 |
| 偏航失準 | 控制 | 偏航電機 | 偏航煞車未完全釋放 | 煞車/齒盤 |
| 發電機振動異常 | 機械 | 變槳軸承 | 軸承內環故障（BPFI 特徵頻率） | 振動頻譜 |
| 變槳系統故障 | 控制 | 液壓油 | 變槳後備電池/超級電容 | 安全停機 |
| 變流器過溫跳脫 | 電氣 | 變流器模組 | 冷卻流量不足/堵塞 | IGBT 散熱 |

每個故障皆含：SCADA 告警 → 4 選 1 診斷測驗 → 5 步 SOP（前兩步預設完成：確認天氣窗、LOTO 上鎖）。

## 6. 主線 7 關攻略（四幕・共用韌體缺陷懸疑）
> 主線是**線性推進**：完成一關才開下一關；過關會推進一天並重擲海象。

| 關 | 幕 | 機組 | 故障（科別／備品） | 獎勵 | 劇情重點 |
|---|---|---|---|---|---|
| 1 序章·走馬上任 | I | CH-12 | 齒輪箱過熱（機械／齒輪箱油） | +15萬 / 100XP | 新人首役，莉莉同行 |
| 2 序章·異狀漸起 | I | CH-07 | 偏航失準（控制／偏航電機） | +16萬 / 110XP | 老船長：告警太密集，怪 |
| 3 第二幕·數據疑雲 | II | CH-21 | 發電機振動（機械／變槳軸承） | +20萬 / 140XP | 監控：多機同時告警非巧合 |
| 4 第二幕·安全警鐘 | II | CH-03 | 變槳故障（控制／液壓油） | +19萬 / 130XP | 工安：先確認後備電源 |
| 5 第三幕·線索浮現 | III | CH-18 | 變流器過溫（電氣／變流器模組） | +17.5萬 / 125XP | 發現：共用同版控制韌體！ |
| 6 第三幕·真相 | III | CH-12 | 齒輪箱過熱（機械／齒輪箱油） | +22萬 / 160XP | 回溯驗證：韌體在特定海象誤動作 |
| 7 終章·核心機組 | IV | CORE | 發電機振動（機械／變槳軸承） | +30萬 / 240XP | 搶修核心、穩住全場併網，通關 |

**通關每關要訣**
1. 接單後先看故障**科別**→ 確認有對應科別技師（沒有就去技師公會招募）。
2. 去交易所買**必備備品**；缺貨會有**到貨前置期**，要預留天數。
3. 看海象：CTV 只能到「警戒」；遇「停航」就「靠港休整」重擲海象，或購 SOV。
4. 出航抵達後，維修畫面：**答對診斷題 + 逐步完成 SOP**，在作業窗倒數內做完（答錯/步驟會消耗作業窗，窗關閉須撤離、可用率小扣）。
5. 完成→可用率回升、領獎勵、圖鑑解鎖→「下一關」。

**通關建議節奏**：前期用調度中心小任務累積現金與技師→備齊各科別技師與常用備品→中段考慮升級工具（降 SOP 耗時）與船舶/SOV（抗海象）→穩定推進到終章。

## 7. 天氣與多日工期（目前實作）
- 每次接單/休整/過關會**隨機擲海象**：約 60% 可作業、30% 警戒、10% 停航。
- 出航需「海象 ≤ 船舶耐受度」；停航時 CTV 不能出航。
- 維修有**作業窗倒數**；天數會因休整、過關、例行任務、備品到貨而前進。

## 8. 排行榜（雲端，目前實作）
- 學生用**暱稱＋班級碼**登入（免密碼），每完成一個任務自動上傳績效分。
- 母港「排行」與「班級動態」讀雲端，顯示全班排名、標示自己。
- 後端為 Google Apps Script Web App（教師自管，免費）。設定見 [LEADERBOARD_SETUP.md](LEADERBOARD_SETUP.md)。

## 9. 建議「修課必須達成目標」（rubric，可自行調整）
> 這是建議草案，數字請依課程調整。
- **基礎（必達）**：通關主線 7 關；圖鑑集滿 5 種故障。
- **進階**：綜合績效分達到 ◯◯ 以上；可用率維持 ≥ 90% 結算。
- **挑戰**：班級排行榜前 ◯ 名；零「天氣硬上/跳步驟」安全事件（待安全計分實作）。

## 10. 設計藍圖（**尚未實作**，用於課程說明與後續開發）
> 以下為規劃方向，目前遊戲**尚未**包含；對應 GitHub Issues 追蹤。

- **多風場營運**：初期管理「單一風場、單一機組工單」；達門檻（足夠資金/人力/船隊）後可**同時營運多個風場**（不同海域、不同機型配置）。
- **可切換主畫面背景**：不同海域、不同風機配置、變電站、船隊背景。
- **多場景登塔**：登船事件（浪高大→船上下起伏、耗時更久、可能登船延誤）、塔架內、機艙內等不同維修場景。
- **突發隨機事件**：人員短缺/罷工、海象導致**交貨延遲**、**定期大修**排程、設備突發故障連鎖、競爭運維商搶單、原廠韌體推送、保固到期等。
- **安全計分**：惡劣海象硬上、跳過 SOP → 安全事件扣分，納入績效。
- **排行榜防作弊與資料保護**（見 Issue）。

## 11. 製作簡報時可用的「三句話介紹」
1. 這是一款把**離岸風電運維**包裝成《大航海時代》式經營遊戲的**教學工具**。
2. 學生透過**故障診斷測驗 + SOP + 天氣/備品/人力的現場限制**，學會真實 O&M 的決策。
