# Handoff：離岸風場運維教學遊戲 — UI 設計參考

## Overview（總覽）
這是一款「離岸風場運維（O&M）教學遊戲」的介面設計，視覺與互動模式參考經典航海／貿易類遊戲
（《大航海時代：傳說》、《大航海時代》、勇者鬥惡龍式的地點選單）。世界觀把航海貿易換算成
離岸風場運維：母港＝運維基地、交易所＝備品市場、商船＝CTV 運維船、商品行情＝風機備品行情、
水手＝運維技師。

目前包含 4 個可互相切換的核心畫面：
1. **母港大廳（Hub）** — 主選單，從這裡前往各設施與出海
2. **備品交易所（Market）** — 風機零件／耗材的買賣與行情
3. **出海航行（Sail）** — CTV 船隊海上航行與船況管理
4. **維修作業（Repair）** — 風機故障診斷與維修教學（含隨堂測驗）

## About the Design Files（關於本設計檔）
本資料夾中的檔案是 **以 HTML 製作的「設計參考稿（design reference）」**，用來呈現預期的
外觀與互動行為，**不是要直接搬進產品的程式碼**。

你的任務是：**在目標專案既有的開發環境中（React / Vue / Svelte / 原生等），沿用其既有的元件、
設計系統與慣例，重新實作這些畫面**。若專案目前還沒有前端環境，請為此專案選擇最合適的框架後再
實作。請把這些 HTML 當成「視覺與行為的規格」，而非要逐字複製的原始碼。

### 如何預覽設計稿
- `運維母港介面.dc.html` 是一個自包含的設計元件（依賴同目錄的 `support.js`，字型由 Google Fonts 載入）。
- 直接用瀏覽器開啟該 `.html` 即可看到完整、可互動的設計（可點頂部分頁切換 4 個畫面、可作答故障診斷測驗）。
- 它是用一個輕量 runtime（`support.js`）渲染的；**請勿把這個 runtime 帶進你的專案**，只需參考其呈現結果。

## Interactions & Behavior（互動與行為）
- **分頁切換**：點頂部 4 個 tab → 切換畫面；當前分頁顯示金色底線。母港的「備」標記與「風場行情」列可點 → Market；出海的「靠泊登塔」鈕 → Repair。
- **故障診斷測驗**：點選任一選項 → 標記 `quiz = index` → 重新計算每個選項樣式與回饋：
  - 正解（index B / 1）：綠邊綠底；回饋綠字「✓ 正確！油位過低或油質劣化使潤滑與散熱下降…」
  - 誤選：該選項紅邊紅底、正解同時顯示綠色、其餘淡化；回饋橘字「✗ 再想想：…應先從潤滑與散熱系統著手。」
- **動畫**：風機葉片 `spin`（9–16s linear 無限）；地點標記 `bob`（4–5s 上下浮動）；告警／海象色點 `shimmer`（1.2–2.4s 透明度脈動）。
- **縮放**：監聽 `resize`，整個 1600×900 舞台等比縮放置中。

## State Management（狀態）
- `screen`：`'hub' | 'market' | 'sail' | 'repair'`（控制畫面切換）
- `quiz`：`null | 0..3`（測驗作答；正解 index = 1）
- `scale`：number（舞台縮放比，由視窗尺寸計算）
- 衍生值：各畫面 display、漲跌顏色（依百分比 ≥100）、星級字串、測驗選項樣式與回饋文字。

## 可調設計參數（原型提供的 Tweaks，可作為元件 props 參考）
- `seaState`：enum `可作業 / 警戒 / 停航` → 控制海象色點與文字（綠/琥珀/紅）
