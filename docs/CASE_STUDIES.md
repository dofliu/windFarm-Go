# 真實案例研究事件 — Real-World Case Studies

> 把過去真實風場「踩過的坑」做成可學習的特殊事件,強化「做中學」。資料見 `src/state/caseStudies.ts`,圖鑑入口為母港「📁 案例檔」。
> 經多來源研究 + 雙重(事實/敏感度)查核後採用。Last updated: 2026-06-25。

## 設計與呈現規則(硬性)
- **named-with-sources(具名+附出處)**:系統性、產業層級、公開紀錄或業界統計類案例 → 遊戲內**顯示真實名稱並附公開出處連結**。
- **anonymized-technical(去識別技術重現)**:可能影射特定在世公司過失、或屬進行中/爭議個案 → **保留完整真實技術情境與 O&M 教訓,但用中性代稱**(如「北美某 ~107m 葉片」「某大型開發商」);`sources` 仍存於資料供維護者查證,但 `visibleSources()` 回空、UI 不顯示具名連結。
- **不臆測**:未公開根因一律以「據報/調查指出/典型而言」限定,不寫成定論;數字均引用自來源並標年份。
- **Tier 漸進**:依運維層級(#76)解鎖,Tier 1 不出現案例(入門保持簡單);複雜/敏感者排到高 Tier。
- **不改主存檔數值**:案例的選項/效果僅在案例檔作教學標示(對/錯 + 為什麼),不直接調整玩家的可用率/預算,避免「真實事故」扭曲個人經營數值。
- **連回既有系統**:每案 `relatesTo` 既有故障(`faults.ts`)或戰情室故障(`incidents.ts`),供圖鑑互跳,不重建內容。

## 在任務清單中自然出現(案例演練 Case Drill)
除了日推進的偶發快報,**自由營運中心(🛰 Ops Center)的任務清單**會以約 1/4 機率抽出「📁 案例演練」,**略過 Tier 解鎖限制**(玩家進中心即可練),優先抽本局未演練過的案例。答題後揭示 O&M 復盤教訓與出處(具名案例),並自動收錄進母港「📁 案例檔」圖鑑;同時把決策對錯記入該科別的「知識點掌握度」。實作:`randomCaseDrill()`(caseStudies.ts)+ `MARK_CASE_SEEN` action + `OpsCenterModal`。

## 維護指引
- **新增案例**:在 `CASE_STUDIES` 加一筆,務必填 `framing`、`minTier`、`relatesTo`、雙語 `title/scenario/lesson/choices`、以及 `sources`(具名案例必附可公開查證的 URL)。
- **去識別案例的紅線**:`title/scenario/lesson/choices` 的可見文字**不得**出現真實風場/OEM/承包商/船名(測試 `test/run.mjs` 的「anonymization guardrail」會以黑名單正則擋下)。
- 測試涵蓋:目錄完整性、schema、Tier 分層單調性、具名必附 URL / 去識別隱藏出處、去識別不洩漏真名、`relatesTo` 連結存在、偶發抽樣的 Tier 閘門與 seen-set。
